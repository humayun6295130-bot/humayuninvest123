Firebase auto-deploy from GitHub (Firestore rules + indexes)
==============================================================

1) On your PC (once), generate a CI token:
   npx firebase-tools login:ci

2) GitHub repo -> Settings -> Secrets and variables -> Actions -> New repository secret
   Name:  FIREBASE_TOKEN
   Value: (paste the token from step 1)

3) Push changes to firestore.rules or firestore.indexes.json on branch "main".
   The workflow "Deploy Firestore" will publish rules + indexes to project in .firebaserc.

If you do NOT add FIREBASE_TOKEN, the workflow skips the deploy job and still passes (no failed check).
You can paste rules from firestore.rules into Firebase Console → Firestore → Rules instead.

Vercel still deploys the Next.js app when you push; it does NOT deploy Firestore.
This workflow keeps Firebase in sync with the repo so you do not have to run firebase deploy manually.

Also add in Vercel (same JSON as CI, or a dedicated key):
  FIREBASE_SERVICE_ACCOUNT_JSON
This enables POST /api/nowpayments/complete-investment so user investments are written reliably after NOWPayments (not only in the browser).
